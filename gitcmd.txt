## Push `proj1` branch content to `main`

If your branch is already created and you want its content in `main`, use this flow:

1. Switch to `main`
   ```bash
   git checkout main
   ```

2. Pull the latest `main` from remote
   ```bash
   git pull origin main
   ```

3. Merge `proj1` into `main`
   ```bash
   git merge proj1
   ```

4. Push `main` to GitHub
   ```bash
   git push origin main
   ```

---

## If `proj1` is not pushed yet

First push the branch:
```bash
git push -u origin proj1
```

Then merge it into `main` as shown above.

---

## What merge does

When you run:
```bash
git merge proj1
```

Git combines the changes from `proj1` into your current branch (`main`).

- If there are no conflicts, Git creates a new merge commit automatically.
- If both branches changed the same part of a file, Git will show a conflict and you must resolve it manually.

---

## Example of a simple merge

```bash
git checkout main
git pull origin main
git merge proj1
git push origin main
```

---

## If you want to avoid a merge commit

You can do a fast-forward merge if `main` has no commits after the branch split:

```bash
git checkout main
git merge --ff-only proj1
```

This is a simpler history, but it only works when possible.
