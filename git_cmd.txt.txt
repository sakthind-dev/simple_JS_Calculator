Git and CLI Commands Used for Debugging
==========================================
git status: Checked repository state and branch.
git log --oneline -10: Viewed recent commit history.
git show --name-only <commit>: Listed files changed in a specific commit.
git show <commit>:<file>: Displayed file content from a specific commit.
git checkout HEAD -- <file>: Retrieved a file from HEAD to working directory.
git ls-tree -r HEAD --name-only: Listed all tracked files in HEAD.
git cat-file -p <commit>: Showed commit details (tree, parents).
git ls-tree <tree>: Listed contents of a tree object.
git show --stat <commit>: Showed commit statistics and file changes.
git ls-files: Listed files in the index.
git show HEAD:<file> | Select-String <pattern>: Searched for secrets in committed file.
New-Item -ItemType Directory -Path <path>: Created directory for file editing.
git show HEAD:<file> | Out-File <file>: Saved committed file to working directory.
git add <files>: Staged modified files.
git commit --amend --no-edit: Amended commit without changing message.
git checkout --orphan <branch>: Created new branch without history.
git add .: Staged all files.
git commit -m "<message>": Committed changes.
git remote add origin <url>: Added GitHub remote.
git push -u origin <branch>: Pushed branch to remote.
